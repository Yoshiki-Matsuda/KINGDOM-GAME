# ログのフル構造化

## 現状

- DB: `game_logs(world_id, idx, message TEXT)` -- メタデータ(ts, actor)も本文も全部テキスト埋め込み
- Rust: `log: Vec<String>` -- 型なし文字列配列
- UI: `parseLogsToHistory()` が正規表現で構造復元 -- 壊れやすい

## 目標データモデル

### DBスキーマ (新テーブル `event_logs`)

```sql
CREATE TABLE event_logs (
  world_id   TEXT NOT NULL,
  id         INTEGER NOT NULL,
  timestamp  INTEGER NOT NULL,
  actor_id   TEXT,
  event_type TEXT NOT NULL,
  data       TEXT NOT NULL DEFAULT '{}',
  message    TEXT NOT NULL DEFAULT '',
  PRIMARY KEY (world_id, id),
  FOREIGN KEY (world_id) REFERENCES worlds(id) ON DELETE CASCADE
);
```

- `event_type`: "battle_start", "attack", "defeat", "loot_gold", "loot_item", "conquest", "conquest_reward", "skill_activation", "explore_complete" 等
- `data`: JSON形式の構造化データ (event_typeごとに異なるフィールド)
- `message`: デバッグ用・フォールバック用の表示テキスト

### Rust 構造体

```rust
// server/src/model/game_event.rs
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct GameEvent {
    pub id: u64,
    pub timestamp: u64,
    pub actor_id: Option<String>,
    pub event_type: String,
    pub data: serde_json::Value,
    pub message: String,
}
```

### GameState 変更

```rust
// Vec<String> -> Vec<GameEvent>
pub log: Vec<GameEvent>,
```

## Task 2: DB マイグレーション作成

**ファイル**: `server/migrations/003_event_logs.sql` (新規)

```sql
CREATE TABLE IF NOT EXISTS event_logs (
  world_id   TEXT NOT NULL,

  timestamp  INTEGER NOT NULL,
  actor_id   TEXT,
  event_type TEXT NOT NULL,
  data       TEXT NOT NULL DEFAULT '{}',
  message    TEXT NOT NULL DEFAULT '',

-- 旧ログのマイグレーション: message列に全文コピー、event_type='legacy'
INSERT INTO event_logs (world_id, id, timestamp, actor_id, event_type, data, message)
SELECT world_id, idx, 0, NULL, 'legacy', '{}', message
FROM game_logs;

DROP TABLE IF EXISTS game_logs;
```

## Task 5: 戦闘ログの構造化 -- combat.rs (最重要・最大ファイル)

**ファイル**: `server/src/model_actions/combat.rs`

主要な push_log/push_actor_log 呼び出し (~60箇所) を構造化ビルダーに置換:

| 旧コード | 新コード |
|---|---|
| `push_actor_log(log, id, "--- 戦利品 ---")` | 削除 (loot イベント自体が構造化) |
| `push_actor_log(log, id, format!("{}を占領！", name))` | `push_conquest_event(log, id, name)` |
| `push_actor_log(log, id, format!("占領報酬: 食料+{}...", ...))` | `push_conquest_reward_event(log, id, food, wood, stone, iron)` |
| `push_log(log, format!("{}が{}に攻撃！({}ダメージ)", ...))` | `push_attack_event(log, None, atk, def, dmg, side)` |
| `push_log(log, format!("{}が{}を撃破", atk, def))` | `push_defeat_event(log, None, atk, def)` |
| `push_actor_log(log, id, format!("{}の演習戦に勝利！", name))` | `push_battle_end_event(log, id, "practice_victory", name)` |

スキル効果ログ (`"  -> ..."`) は `push_system_event` で段階的移行。

```typescript
  switch (event.event_type) {
    case "attack":
      return renderAttackHtml(event.data);  // data: {attacker, target, damage, side}
    case "loot_gold":
      return renderResourceDeltaHtml("gold", event.data.amount);
    case "conquest_reward":
      return renderConquestRewardHtml(event.data);  // data: {food, wood, stone, iron}
    // ...
  }
}
```

## Task 8: legacy イベントの後方互換

`event_type: "legacy"` のイベント（旧ログからマイグレーションされたもの）は、`message` フィールドを従来のテキストパーサーで表示するフォールバックパスを残す。新ログが完全に構造化された後、段階的に legacy パーサーを削除。
